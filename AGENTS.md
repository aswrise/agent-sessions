## Agent skills

### Issue tracker

GitHub Issues；外部 PR 不作为需求入口。见 `docs/agents/issue-tracker.md`。

### Triage labels

使用默认五状态标签。见 `docs/agents/triage-labels.md`。

### Domain docs

Single-context：根目录 `CONTEXT.md`，架构决策放 `docs/adr/`。见 `docs/agents/domain.md`。

### Plans and TODOs

计划写入仓库内 `plans/`，待办写入仓库内 `todos/`。不要把本项目的计划或待办写到仓库外；需要跨电脑继续的文档必须 commit 并 push。
