首页「有链」状态 与 整链悬浮预览

针对 TODO 2 / TODO 3 各给三个方向，全部用现有 dashboard 的色板、字体和浮层规则渲染成可交互 mockup（徽标可 hover / 点击 / 键盘 focus）。方案之间可以自由组合，最后有推荐组合和待确认问题。

只读 /api/lineages 缓存，不逐行请求 1900+ 行无性能回退 不只靠颜色表达状态 不与首条消息 Transcript 预览抢浮层 大链（58 节点）不可压成缩略图

TODO 2首页列表：怎么表示「有链」

列表已有 13 列，信息密度接近上限。三个方案按信息量从多到少排列；无链行完全不渲染任何元素，是三案共同前提。

方案 A

名称旁链徽标 — 不加列，表达「有链 + 整链规模」

在名称文字后放一枚胶囊徽标：链形 glyph + 整链 Session 数。徽标是独立按钮（stopPropagation），不影响点击名称进入改名；hover / focus 徽标触发 TODO 3 的链预览。

TODO 3悬浮预览整条链：浮层形态与大链策略

共同前提：预览锚定在 TODO 2 的链徽标上（不是整行），与首条消息的 Transcript 预览分属不同触发区，互不抢占；沿用现有 tip 的节奏 — 180ms 延迟出现、100ms 延迟消失、浮层自身可进入（mouseenter 取消关闭）。上面方案 A 的徽标已经演示了「迷你 DAG」形态，这里补齐另外两种形态和大链降级。

方案 1

锚定 Hover Card · 迷你 DAG（≤12 节点原样画）

复用 LineageGraph 的分层布局数据，节点从 220px 缩到 150px，去掉 id 行。边为有向边（箭头指向下游），列间距固定 80px，文件名标签放在列间空隙中，永不被节点遮挡。小链空间关系一目了然，和关系图页视觉同源 — 预览就是关系图的缩小版，点过去不需要重新建立心智模型。演示见上方方案 A 的徽标。

优点

  • 分叉 / 汇聚等拓扑信息不丢；边上还能标传递的文件名
  • 与全局关系图视觉连续，「在关系图中定位」的跳转不突兀

代价

  • 超过 ~12 节点必须降级（见下方大链策略），实现要维护两套渲染分支
  • 浮层宽度随链的层数增长，需要 max-width + 内部横向滚动兜底
方案 2

上下游轴线摘要 — 以当前 Session 为中心的三栏清单

不画图，按「上游 / 当前 / 下游」三栏列出直接邻居（各栏最多 4 项，多余折叠成计数）。永远不会溢出，天生适配任何链规模，代价是丢掉隔层拓扑。

大链策略

邻域裁剪：超过 12 节点只画「当前 ±1 层」，其余按上游 / 下游 / 旁支折叠

方案 1 / 2 共用的降级规则。折叠节点可点击 — 直接跳到全局关系图并定位当前 Session；本例没有兄弟旁支，所以只显示上游和下游两个计数节点。58 节点的链永远不会被塞进浮层。

通用键盘 · 触屏 · Reduced-motion 等价入口

三个 TODO 3 方案共用同一套输入契约，锚点都是链徽标（真按钮，进 Tab 序）。

输入触发保持 / 关闭
鼠标hover 徽标 180ms 后出现（沿用现有 tip 节奏）；点击徽标 = 固定浮层移入浮层取消 100ms 关闭计时；固定后点外部或 Esc 关闭
键盘Tab 聚焦徽标即显示预览；Enter 固定并把焦点移入浮层浮层内 Tab 循环节点，Esc 关闭并把焦点还给徽标
触屏无 hover：tap 徽标直接等于「固定」，二段确认后 tap 节点才导航tap 浮层外关闭；行点击复制的既有手势不受影响（徽标 stopPropagation）
Reduced-motion去掉 scale/translate，仅保留 ≤140ms 的 opacity 显隐迷你 DAG 的节点入场 stagger 整体停用

动效基线：浮层 140ms cubic-bezier(.23,1,.32,1)，transform-origin 对准徽标；徽标按压 scale(.95)。悬浮预览一天要看几十次，不做入场 stagger、不超过 200ms。

收束推荐组合与待确认问题

组合构成评价
推荐 TODO 2 方案 A（名称旁徽标）+ TODO 3 方案 1（迷你 DAG）+ 大链邻域裁剪 + 方案 C 的「有链」tab 作筛选补充 零新列、一个锚点承担显示 / 预览 / 固定三职责；小链所见即拓扑，大链有明确出口；tab 解决批量发现
备选 TODO 2 方案 B（独立列）+ TODO 3 方案 2（轴线摘要） 如果你更看重「列表内直接看到角色 + 可排序」，且接受 +1 列；实现量最小的一档
重型 TODO 2 方案 A + TODO 3 方案 3（侧边面板） 如果大链是主要使用场景（58 节点链会常看），面板一步到位，但偏离「悬浮预览」的原始诉求
  1. 行内状态的信息量拍板：只要「有链」，还是要整链数（A）/ 角色+上下游数（B）？这决定 TODO 2 选型，也决定 aria-label 和排序能力。
  2. 点击徽标的语义：固定预览（推荐）还是直接跳全局关系图定位？推荐固定预览 + 浮层 footer 提供「在关系图中定位」，跳转始终多一步但不易误触。
  3. 「有链」筛选要不要进 tabs：tabs 已有 6 项；不进 tabs 的替代是高级筛选区加一个开关。
  4. 大链阈值：迷你 DAG 原样渲染的上限定多少？建议 12（约两屏节点宽度内），也可以按浮层可用宽度动态算。
  5. 预览内点节点的行为：直接打开详情（推荐，与关系图页一致），还是先在预览内切换焦点节点？
